🌐 VLAN Segment Manager - Help & Documentation

Authentication required for changes: Every write operation (create, update, delete, allocate, release) requires logging in first. Open the Swagger UI (/docs), call POST /api/auth/login with {"username": "admin", "password": "admin"}, then use "Try it out" on the endpoints below — Swagger keeps the session cookie for you.

➕ How to Add a New Segment

Use the Swagger UI to call:

POST /api/segments - Create a new segment
{ "site": "site1", "vlan_id": 150, "epg_name": "EPG_WEB", "segment": "192.168.15.0/24", "dhcp": false }

Notes:

  • vlan_id must be unique within the site (1-4094)
  • segment must be a valid network address (e.g. 192.168.15.0/24, not a host address) and must start with the site's configured IP prefix (e.g. site1 → 192.x.x.x)
  • epg_name must be unique for the same VLAN ID at a site
  • Subnet mask must be between /16 and /29

📌 How to Allocate a Segment

Allocation automatically finds an available segment at the site and assigns it to the given cluster name.

POST /api/allocate-vlan - Allocate a segment for a cluster
{ "cluster_name": "web-cluster", "site": "site1" }

If the cluster already has a segment at that site, the existing segment is returned instead of allocating a new one. If no segments are available, the request fails — create more segments first.

Shared Segments

To assign multiple clusters to the same segment, allocate it to one cluster first, then update the cluster list directly:

PUT /api/segments/{segment_id}/clusters - Update cluster assignment
{ "cluster_names": "frontend,backend,database" }

🗑️ How to Delete a Segment

First find the segment's ID, then delete it. A segment can only be deleted if it is not currently allocated.

GET /api/segments/search?q=EPG_WEB - Find the segment ID
DELETE /api/segments/{segment_id} - Delete the segment
Note: If the segment is allocated, deleting it fails. Deallocate it first (see below).

🔓 How to Deallocate a Segment

Releasing frees the segment so it can be allocated to another cluster.

POST /api/release-vlan - Release a cluster's segment
{ "cluster_name": "web-cluster", "site": "site1" }

For a shared segment, this only removes the specified cluster from the assignment — other clusters remain allocated. To release all clusters from a shared segment at once, set an empty cluster list via the clusters endpoint:

PUT /api/segments/{segment_id}/clusters - Release all clusters
{ "cluster_names": "" }

🏗️ How to Add a New Site (in code)

Sites are configured entirely via environment variables — no code changes are needed for a basic new site.

  1. Add the new site name to the SITES environment variable (comma-separated list), e.g. in .env:
    SITES="site1,site2,site3,site4"
  2. Add a corresponding entry to SITE_PREFIXES giving the IP prefix (first octet) used to validate segments for that site:
    SITE_PREFIXES="site1:192,site2:193,site3:194,site4:195"
  3. Restart the application.
Fail-fast validation: On startup, validate_site_prefixes() (in src/config/settings.py) checks that every site in SITES has a matching entry in SITE_PREFIXES. If a site is missing its prefix, the app crashes immediately with a clear error message rather than allowing inconsistent runtime behavior.

Once configured, the new site will automatically appear in /api/sites, the site filter dropdown, and the statistics dashboard. Any segment created for that site must use a CIDR starting with its configured prefix.

🔧 Troubleshooting

Common Issues

VLAN ID Already Exists

Error: "VLAN X already exists for site Y"

Solution: Each VLAN ID must be unique within a site. Choose a different VLAN ID or edit the existing segment.

Invalid IP Prefix

Error: "Invalid IP prefix for site. Expected to start with X"

Solution: Ensure your segment starts with the correct prefix for the site:

  • site1: 192.x.x.x
  • site2: 193.x.x.x
  • site3: 194.x.x.x

No Available Segments

Error: "No available segments"

Solution: Create new segments for the site or release existing allocated segments.

Cannot Delete Allocated Segment

Error: Segment deletion fails

Solution: Release the segment first, then delete it.

Best Practices

  • Use descriptive EPG names
  • Plan your VLAN ID ranges per site to avoid conflicts
  • Use shared segments sparingly and document which clusters share them
  • Regularly export data for backup purposes
  • Monitor the statistics to track utilization

Support

For additional support or to report issues:

  • Check the application logs for detailed error messages
  • Use the /api/health endpoint to verify system status
  • Consult the Swagger UI (/docs) for full request/response formats
